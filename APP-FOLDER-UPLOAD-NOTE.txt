AR Studio app folder upload note

I inspected D:\AR Studio\app.

Findings:
- The app folder is small: 78 files, about 191 KB.
- There are no generated folders inside app.
- The unused app/api/auth/[...nextauth] placeholder route was removed because it can confuse GitHub web upload.
- The remaining bracket folders are required Next.js routes, for example [id], [businessSlug], [productSlug], and [tableNumber]. Do not rename them.

If GitHub web upload still fails:
- Use GitHub Desktop or git command line instead of browser drag-and-drop.
- Git supports these folder names correctly.

Recommended command flow:
1. Clone/open your GitHub repo locally.
2. Copy D:\AR Studio\app into the repo root so it becomes repo\app.
3. Run:
   git add app
   git commit -m "Add updated AR Studio app folder"
   git push

Do not upload:
- node_modules
- .next
- .env
- .env.local
